## ? 학습 진행 모니터링

### 1. 게임 창에서 실시간 확인

학습이 시작되면 게임 창이 자동으로 열립니다:
- **실시간 게임 플레이** 시각화
- **봇 동작** 실시간 확인
- **전투 상황** 모니터링

### 2. 로컬 모니터링 서버에서 통계 확인

#### 모니터링 대시보드 접속

- **URL**: `http://localhost:8000`
- **자동 시작**: 학습 시작 시 모니터링 서버 자동 시작
- **브라우저**: 자동으로 대시보드 열림

#### 확인 가능한 통계

1. **게임 통계**
   - 총 게임 수
   - 승리/패배 수
   - 승률
   - 평균 게임 시간

2. **학습 통계**
   - 학습 진행률
   - 모델 성능
   - 빌드 오더 파라미터

3. **실시간 상태**
   - 현재 게임 상태
   - 전투 통계
   - 리소스 사용량

## ? 학습 시작

### 실행 방법

```batch
bat\training_with_monitoring_and_auto_learning.bat
```

### 학습 설정

- **모드**: 로컬 트레이닝 (Single instance)
- **시각화**: 활성화 (게임 창 표시)
- **모니터링**: 자동 시작
- **신경망**: 활성화

---

## ? 학습 완료 후 자동 워크플로우

학습이 완료되면 (또는 Ctrl+C로 중단하면) 자동으로 다음 작업이 실행됩니다:

### STEP 1: 리플레이 비교 분석 실행

- 프로 리플레이 vs 훈련 리플레이 비교
- 차이점 파악
- 개선 사항 식별

### STEP 2: 프로 리플레이에서 재학습

- 프로 게이머 리플레이 분석
- 최적 빌드 오더 학습
- 전략 추출

### STEP 3: 학습 파라미터 적용

- 학습된 파라미터 저장
- 다음 훈련에 자동 적용
- 빌드 오더 최적화

## ? 워크플로우 요약

```
1. 학습 시작
   ↓
2. 모니터링 서버 자동 시작
   ↓
3. 게임 창 열림 (실시간 시각화)
   ↓
4. 대시보드 열림 (통계 확인)
   ↓
5. 학습 진행 (게임 창 + 대시보드 모니터링)
   ↓
6. 학습 완료 (또는 Ctrl+C)
   ↓
7. 리플레이 비교 분석 자동 실행
   ↓
8. 프로 리플레이에서 재학습
   ↓
9. 학습 파라미터 적용
   ↓
10. 다음 훈련 준비 완료
```

## ? 사용 팁

### 모니터링

1. **게임 창**: 실시간 게임 플레이 확인
2. **대시보드**: 통계 및 성능 확인
3. **로그**: 콘솔 출력에서 상세 정보 확인

### 학습 중단

- **Ctrl+C**: 학습 중단 후 자동으로 post-learning 워크플로우 실행
- **게임 종료**: 자연스럽게 종료되면 자동으로 post-learning 실행

### 다음 학습

- 학습 파라미터가 자동으로 적용되어 다음 학습에 사용됩니다
- 개선된 빌드 오더로 더 나은 성능을 기대할 수 있습니다
